# 손·손가락 SVG 기반 B♭ 트럼펫 계이름 운지표 업그레이드 바이브코딩 프롬프트

이 문서는 `01 Source` 폴더의 자료를 분석해, 손과 손가락이 실제 밸브를 누르는 모습을 정교한 SVG로 보여 주는 B♭ 트럼펫 계이름 운지표를 다시 만드는 실행용 프롬프트다. 아래의 **붙여넣기용 프롬프트 전체**를 코딩 에이전트에게 그대로 제공하면 된다.

## 사용 전제

- 작업 루트는 현재 저장소다.
- `01 Source`는 읽기 전용 원본으로 취급한다.
- 최종 결과는 외부 네트워크 없이 동작해야 한다.
- 결과물은 화면용 인터랙티브 HTML, 독립 정적 SVG, A4 가로 PDF의 세 형식이다.
- 계획만 쓰고 끝내지 말고 파일 생성, 자동 검사, 실제 렌더링 검수까지 완료해야 한다.

## 붙여넣기용 프롬프트 전체

### 0. 역할과 최종 목표

당신은 다음 다섯 역할을 동시에 수행한다.

1. 트럼펫 교육 콘텐츠 설계자
2. 해부학적 손 SVG 일러스트레이터
3. SVG 모션 엔지니어
4. 접근성 중심 프론트엔드 개발자
5. 인쇄·PDF 프리프레스 검수자

현재 작업공간의 `01 Source` 폴더와 루트의 `트럼펫 계이름 운지법 애니메이션 웹앱 개발서.md`를 먼저 읽고, B♭ 트럼펫 기보음 F♯3-C6의 31음을 다루는 계이름 운지표를 업그레이드하라.

새 차트의 핵심은 단순한 빨강·흰색 밸브 원에 머무르지 않고, 다음을 동시에 보여 주는 것이다.

- 오선 위 음표
- 기보음과 이명동음
- 한국어 고정도법 계이름
- 오른손 검지·중지·약지의 실제 대응
- 손가락이 피스톤 캡을 누르고 떼는 위치 변화
- 표준 주운지와 대표 대체 운지
- B♭ 트럼펫의 실음

작업 완료 시 다음 네 파일을 생성하라.

```text
output/
├─ trumpet-fingering-chart-vibe-coding-prompt-ko.md
├─ trumpet-fingering-chart-hand.html
├─ trumpet-fingering-chart-hand.svg
└─ pdf/
   └─ trumpet-fingering-chart-hand.pdf
```

기존 `01 Source` 파일은 절대 수정하지 마라. 임시 파일은 `tmp/` 아래에만 둔다.

### 2. 음악 데이터의 단일 진실 공급원을 만들어라

HTML, SVG, PDF에 각각 별도의 운지 데이터를 복사하지 마라. 다음 구조의 `NOTES` 배열 하나에서 모든 형식을 생성하라.

```js
{
  midi: 60,
  name: "C4",
  enh: null,
  solfege: "도",
  valves: [],
  alts: [],
  concert: "B♭3",
  register: "low",
  staffY: 0
}
```

데이터 규칙:

- 정확히 31개다.
- MIDI는 54부터 84까지 중복과 누락 없이 연속이다.
- `valves`와 각 `alts`는 `1`, `2`, `3`만 포함하며 오름차순이다.
- 빈 배열은 개방이며 화면에는 `0 (개방)`으로 표시한다.
- 계이름은 B♭ 실음이 아니라 **기보음 기준 고정도법**이다.
- 이명동음은 `F♯3 / G♭3`처럼 함께 표시한다.
- 실음 MIDI는 `writtenMidi - 2`다.
- 음 조회 키는 계이름 문자열이 아니라 MIDI 또는 고유 ID다.

#### 2.2 로컬 대체 운지 두 건의 보정

로컬 `bb_trumpet_standard_fingering_chart.html`에는 다음 두 대표 대체 운지가 들어 있지만, 새 결과물에서는 제외한다.

- D4의 대체 `3`
- F♯5의 대체 `1-3`

이 둘은 Yamaha 공식 인터랙티브 운지표와 대조했을 때 대표 대체 운지로 확인되지 않는다. 원본 파일은 수정하지 않고, 새 단일 데이터 모델에만 보정을 적용한다. 보정 내역은 최종 보고에 남긴다.

참고 링크:

- Yamaha 공식 운지표: <https://www.yamaha.com/en/musical_instrument_guide/trumpet/play/play002.html>
- Yamaha 공식 운지표 PDF: <https://www.yamaha.com/en/musical_instrument_guide/common/images/trumpet/fingering.pdf>
- Yamaha 밸브·배음 원리: <https://www.yamaha.com/en/musical_instrument_guide/trumpet/mechanism/mechanism002.html>

### 3. 세 출력물의 구조를 한 번만 설계하라

```text

      ↓

      ├─ 독립 .svg 저장
      ├─ HTML의 인쇄 전용 영역에 동일 마크업 삽입
```

세 파일을 손으로 따로 편집하지 마라. 변경할 때마다 데이터와 정적 차트는 생성 스크립트로 다시 만든다.


- 미니 손·밸브 포즈 생성
- canonical SVG 생성
- HTML 생성
- 출력 디렉터리 생성

### 4. 인쇄용 canonical SVG를 먼저 완성하라

루트 요소 계약:

```html
<svg xmlns="http://www.w3.org/2000/svg"
     width="297mm"
     height="210mm"
     viewBox="0 0 1600 1131"
     role="img"
     aria-labelledby="chart-title chart-desc">
```

필수 요소:

- `<title>`: 손가락으로 익히는 B♭ 트럼펫 계이름 운지표
- `<desc>`: 음역, 31음, 계이름, 손가락, 주운지·대체 운지를 설명하는 완결 문장
- `<metadata>`: 로컬 소스, 대체 운지 보정, Bravura 라이선스 정보
- 외부 이미지, 외부 CSS, 외부 폰트 URL, `foreignObject`, 실행 스크립트 없음

#### 4.1 레이아웃 좌표

다음 영역을 기준으로 구성하라.

| 영역 | y 범위 | 내용 |
|---|---:|---|
| 헤더 | 0-205 | 제목, 음역, 범례, 큰 오른손-밸브 안내 그림 |
| 낮은 음역 | 208-468 | F♯3-F4, 12음 |
| 중간 음역 | 482-742 | F♯4-F5, 12음 |
| 높은 음역 | 756-1016 | F♯5-C6, 7음 |
| 푸터 | 1034-1120 | 읽는 순서, 이조, 배음, 출처, 라이선스 |

각 음역 패널은 다음을 갖는다.

- 라운드 패널 배경
- 금색 상단 구분선
- 한국어 음역명, 영어 음역명, 음역 범위
- 5선 오선과 높은음자리표
- 음별 세로 구분선
- 음표와 필요한 보조선

각 음 셀에는 다음 순서로 넣는다.

1. 오선 위 음표
2. 기보음과 이명동음
3. 한국어 계이름
4. 미니 오른손·세 밸브 SVG
5. 숫자 주운지
6. 대체 운지 칩 또는 `표준 운지`

#### 4.2 오선 위치 계산

음표 위치를 음마다 임의 좌표로 하드코딩하지 마라. 문자 이름과 옥타브로 온음계 인덱스를 만든다.

```js
const LETTER_INDEX = { C:0, D:1, E:2, F:3, G:4, A:5, B:6 };
const diatonicIndex = octave * 7 + LETTER_INDEX[letter];
```

높은음자리표의 아래 첫째 줄 E4를 기준으로 반 칸씩 이동한다. 낮은음은 C4, A3, F3 위치에 보조선을 추가하고, 높은음은 A5, C6 위치에 보조선을 추가한다.

행별 음역이 다르므로 오선의 세로 중심을 조정해 다음 결함을 막는다.

- F♯3 음표와 `F♯3 / G♭3` 라벨의 충돌
- C6 보조선과 음역 제목의 충돌
- 샤프와 이전 셀 음표의 충돌

높은음자리표 path를 기존 자료에서 재사용하면 SVG `<metadata>`와 푸터에 다음 취지의 저작자 표시를 유지한다.

```text
음표 글리프: Bravura · © Steinberg Media Technologies GmbH · SIL OFL 1.1
```

#### 4.3 인쇄 가독성 합격 조건

- A4 가로 실물 출력에서 핵심 계이름과 운지 숫자가 읽힌다.
- 최소 안전 여백은 약 8mm다.
- 셀의 텍스트, 음표, 손, 칩이 겹치지 않는다.
- 누른 상태는 색 외에도 손가락 높이, 캡 높이, 숫자로 구분된다.
- 흑백으로 출력해도 밸브 번호와 눌림 위치를 이해할 수 있다.
- `F♯3`, `C4`, `C♯4`, `E5`, `C6` 셀을 200% 확대해도 선이 선명하다.

### 5. 큰 손·트럼펫 SVG를 해부학적으로 설계하라

화면용 큰 장면은 관객이 트럼펫을 옆에서 보는 방향이다.

- 벨: 왼쪽
- 마우스피스: 오른쪽
- 3번 밸브: 왼쪽, 벨에 가장 가까움
- 2번 밸브: 가운데
- 1번 밸브: 오른쪽, 마우스피스에 가장 가까움

루트 개발서의 기본 좌표 계약을 우선 사용한다.

```text
viewBox: 0 0 720 520 또는 같은 비율의 확장 좌표계
1번 밸브 중심 x=402
2번 밸브 중심 x=342
3번 밸브 중심 x=282
캡 중심 y=210
눌림 시 피스톤 이동 +12px
```

더 큰 자체 좌표계를 쓰면 비율을 유지한 대응표를 주석으로 남긴다.

```text
   ├─ #scene-rh-palm
   ├─ #scene-rh-thumb
   ├─ #scene-rh-pinky
   ├─ #scene-finger-index
   ├─ #scene-finger-middle
   └─ #scene-finger-ring
```

#### 5.2 오른손 해부학 조건

- 검지 끝은 1번 캡 중앙과 만난다.
- 중지 끝은 2번 캡 중앙과 만난다.
- 약지 끝은 3번 캡 중앙과 만난다.
- 소지는 밸브를 누르지 않고 핑거 후크에 자연스럽게 걸린다.
- 엄지는 1·2번 밸브 사이 리드파이프 아래를 지지한다.
- 손목은 과도하게 꺾이지 않는다.
- 손가락이 서로 교차하거나 관을 뚫지 않는다.
- 눌린 손가락의 손끝 패드와 캡 사이에 눈에 띄는 공백이 없어야 한다.

각 움직이는 손가락은 가능하면 다음 중첩 구조를 가진다.

```text
.finger-root
└─ .seg-prox
   └─ .seg-mid
      └─ .seg-distal
         ├─ .fingertip-pad
         ├─ .nail
         ├─ .crease
         └─ .contact-anchor
```

리얼리즘 표현:

- 손바닥과 손가락에 동일한 피부 그라디언트 사용
- 외곽선은 피부보다 한 단계 어두운 중립 갈색
- 손등의 밝은 면에 얇은 하이라이트
- PIP·DIP 관절에 짧고 완만한 주름
- 손톱은 손끝보다 밝고 채도 낮게
- 손가락 사이 가림 관계를 레이어 순서로 표현
- 그림자는 교육 정보를 방해하지 않는 낮은 불투명도
- 래스터 손 사진을 넣지 말고 전체를 SVG path로 작성

#### 5.3 왼손 파지 조건

- 왼손 손바닥은 밸브 케이싱 아래를 감싼다.
- 왼손 엄지는 케이싱 반대쪽을 지지한다.
- 왼손 약지는 3번 밸브 슬라이드 링에 연결된다.
- 저음의 1-3 또는 1-2-3 운지에서만 선택적으로 슬라이드 보정 모션을 보여 준다.

#### 5.4 손-악기 시각 합격 조건

- 손의 크기가 밸브 간격과 비교해 비현실적으로 작거나 크지 않다.
- 검지·중지·약지의 길이 차이가 느껴진다.
- 모든 손가락이 같은 굵기의 막대처럼 보이지 않는다.
- 캡이 손가락 위에 그려지는 레이어 오류가 없다.
- 손가락 끝이 캡 중심에서 좌우 3px 이상 벗어나지 않는다.
- 손끝과 캡의 눌림 완료 오차는 2px 이하를 목표로 한다.

### 6. 8개 운지 포즈를 명시적으로 지원하라

논리 키는 밸브 1·2·3 순서의 비트 문자열이다.

```js
function poseKey(valves) {
  return [1, 2, 3].map(v => valves.includes(v) ? "1" : "0").join("");
}

poseKey([])       === "000";
poseKey([1])      === "100";
poseKey([2])      === "010";
poseKey([3])      === "001";
poseKey([1,2])    === "110";
poseKey([1,3])    === "101";
poseKey([2,3])    === "011";
poseKey([1,2,3])  === "111";
```

주의: `poseKey`는 논리적 밸브 순서 1-2-3이고, 옆면 장면의 화면상 위치는 왼쪽부터 3-2-1이다.

인쇄 셀의 미니 손은 주운지를 기본 그림으로 사용한다. 대체 운지는 텍스트 칩으로 표시한다. HTML에서 대체 운지를 선택하는 기능을 추가한다면 큰 장면만 해당 대체 포즈로 전환한다.

미니 손 포즈 조건:

- 손바닥, 세 손가락, 세 캡, 세 케이싱이 보인다.
- 누른 손가락은 캡과 함께 아래로 이동한다.
- 누른 캡은 빨강, 놓은 캡은 흰색을 기본으로 하되 숫자를 항상 표시한다.
- 색을 제거해도 손가락과 캡의 세로 위치로 상태를 구분할 수 있다.
- 모든 셀에 중복 SVG ID를 만들지 않는다.

다음 대표 음으로 8개 포즈를 검사한다.

| 포즈 | 대표 음 또는 대체 운지 |
|---|---|
| 000 | C4, G4, C5, E5 |
| 100 | F4, D5 |
| 010 | B3, F♯4, B4 |
| 001 | A3 대체, E5 대체 |
| 110 | A3, E4, A4 |
| 101 | G3, D4 |
| 011 | G♯3, D♯4, G♯4 |
| 111 | F♯3, C♯4 |

### 7. 손가락과 피스톤 모션을 동기화하라

기본 타이밍:

| 대상 | 정지 → 눌림 | 누름 | 복귀 | 이징 |
|---|---|---:|---:|---|
| 피스톤 그룹 | `translateY(0 → 12px)` | 110ms | 150ms | 누름 `cubic-bezier(.3,0,.2,1)`, 복귀 `cubic-bezier(.34,1.3,.3,1)` |
| 검지 | 회전 약 `0 → -10deg` | 110ms | 150ms | 피스톤과 동일 |
| 중지 | 회전 약 `0 → -6deg` | 110ms | 150ms | 피스톤과 동일 |
| 약지 | 회전 약 `0 → -4.5deg` | 110ms | 150ms | 피스톤과 동일 |

좌표계를 변경했다면 손가락의 피벗-손끝 수평거리로 각도를 다시 계산한다.

```text
필요 회전각 ≈ asin(피스톤 이동량 / 피벗-손끝 거리)
```

모션 규칙:

- 한 음의 여러 밸브는 같은 프레임에서 동시에 움직인다.
- JS는 상태 클래스만 토글하고 CSS transition이 보간한다.
- 빠른 연타에서는 큐를 쌓지 않고 마지막 선택 상태가 이긴다.
- 손가락과 캡은 같은 타이밍과 이징을 사용한다.
- 애니메이션 속성은 `transform`, `fill`, `stroke`, `opacity`로 제한한다.
- 눌린 캡의 색만 바꾸고 위치를 그대로 두는 가짜 모션은 금지한다.
- 손가락만 움직이고 피스톤이 멈춰 있는 상태도 금지한다.
- `prefers-reduced-motion: reduce`에서는 즉시 최종 포즈로 바꾸되 정보는 동일하게 유지한다.

3번 슬라이드 보정:

- 대상: 낮은 음역의 1-3, 1-2-3 조합
- `#scene-slide-3`과 `#scene-lh-ring`을 함께 `translateX(-14px)`
- 시간: 220ms ease-out
- 고음 대체 운지에는 자동 적용하지 않는다.

### 8. HTML을 반응형 인터랙티브 차트로 만들어라

HTML은 단일 파일이어야 한다.

- 인라인 CSS
- 인라인 SVG
- 인라인 JavaScript
- 런타임 외부 요청 0건
- 로컬 파일로 직접 열어도 동작

화면 구조:

```text
헤더
├─ 제목과 설명
└─ 기보음/실음, 음 재생, A4 인쇄

주 작업영역
├─ 큰 트럼펫+양손 SVG
└─ 현재 음 상세 카드
   ├─ 계이름
   ├─ 기보음·이명동음
   ├─ 손가락 이름
   ├─ 밸브 1·2·3 상태
   ├─ 대표 대체 운지
   └─ 실음

31음 선택 영역
├─ 낮은 음역 12개
├─ 중간 음역 12개
└─ 높은 음역 7개
```

초기 선택은 D4를 권장한다. 검지와 약지가 함께 움직여 1번·3번의 물리적 위치를 첫 화면에서 쉽게 확인할 수 있기 때문이다.

음 선택 버튼에는 다음을 표시한다.

- 계이름
- 기보음·이명동음
- 세 개의 밸브 상태
- 완결된 `aria-label`

상호작용:

- 버튼 클릭 또는 Enter/Space: 음 선택
- 좌우 화살표: 이전/다음 음
- Home/End: 첫 음/마지막 음
- `선택 음 재생`: 사용자 동작 후 Web Audio로 약 1.2초 재생
- `기보음/실음`: 재생 MIDI만 전환
- `A4 차트 인쇄`: `window.print()`
- 음 선택만으로 자동 재생하지 않아도 된다. 예기치 않은 소리를 피하려면 별도 재생 버튼이 낫다.

Web Audio 기준:

```js
const midi = pitchMode === "concert" ? note.midi - 2 : note.midi;
const frequency = 440 * Math.pow(2, (midi - 69) / 12);
```

새 음을 재생하면 이전 오실레이터를 짧게 페이드아웃하고 중첩 클리핑을 막는다.
